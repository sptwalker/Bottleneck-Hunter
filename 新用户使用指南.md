## 1. 环境准备与安装

**要求**：Python 3.10+

```bash
# 进入项目目录后，可编辑方式安装（会注册 bottleneck-hunter 命令）
pip install -e .
```

安装后即可使用 `bottleneck-hunter` 命令行工具。

---

## 2. 配置 API Key（.env）

本系统靠大语言模型（LLM）驱动分析，**至少需要配置一个可用的模型服务商 Key**。

```bash
# 复制模板
cp .env.example .env
```

编辑 `.env`，填入你有的服务商 Key（**填你要用的即可，不必全填**）：

| 服务商 | .env 变量 |
|---|---|
| OpenAI | `OPENAI_API_KEY` |
| Anthropic (Claude) | `ANTHROPIC_API_KEY` |
| Google (Gemini) | `GOOGLE_API_KEY` |
| DeepSeek | `DEEPSEEK_API_KEY` |
| 通义千问 Qwen | `DASHSCOPE_API_KEY` |
| 智谱 GLM | `ZHIPU_API_KEY` |
| MiniMax | `MINIMAX_API_KEY` |
| OpenRouter | `OPENROUTER_API_KEY` |
| SiliconFlow | `SILICONFLOW_API_KEY` |
| Agnes | `AGNES_API_KEY` |
| Kimi (月之暗面) | `MOONSHOT_API_KEY` |

其它常用项：
- `OUTPUT_LANGUAGE=zh` — 输出中文（默认）
- `LOG_LEVEL=INFO` — 日志级别
- `SMTP_*`（可选）— 注册/改邮箱的验证码邮件发送；**不配则验证码直接打印到服务器日志**（开发/内网够用）

> 💡 API Key 也可以启动后在 **系统配置中心 → API Key 管理** 里按服务商填写并持久化（按用户隔离）。`.env` 里放一个能用的 Key 先把系统跑起来即可，其余在网页里配更方便。

## 3. 启动服务器

```bash
bottleneck-hunter serve
# 默认监听 http://127.0.0.1:8000
```

可选参数：

```bash
bottleneck-hunter serve --port 8000 --host 127.0.0.1
# 局域网访问：--host 0.0.0.0
```

启动成功后终端会显示访问地址，浏览器打开即可。

> ⚠️ 必须用 `bottleneck-hunter serve` 启动，`python -m web.app` **不会**起服务。

## 4. 首次登录与账户

### 4.1 默认管理员（首次启动自动创建）
系统**首次启动且无任何用户**时，会自动创建管理员账户：
- 用户名：`admin`
- 密码：**随机生成，仅在服务器日志里打印一次**，查找日志中的关键字：
  ```
  ⚠️  已创建默认管理员 admin，临时密码: xxxxxxxx（仅显示一次，请尽快修改）
  ```
- 用这个账户登录后，**请立刻到账户设置里改密码**。

### 4.2 普通用户注册
在登录页可注册新用户，流程通常为：
1. 填写用户名/邮箱/密码，可能需要**邀请码**（由管理员在「用户管理中心 → 邀请码」生成）；
2. 系统发送**邮箱验证码**——若未配置 SMTP，则验证码打印在服务器日志里；
3. 输入验证码完成注册。

### 4.3 账户设置
右上角头像 → 账户设置：改显示名、邮箱、密码。旁边的按钮可退出登录。

## 5. 首次配置向导：系统配置中心（必做）

登录后**第一件事**：进入顶栏 **系统配置中心**，把模型配好，否则分析会提示「无可用 LLM」。

系统配置中心分几块：

### 5.1 API Key 管理（Provider 统一管理）
- 每个服务商（OpenAI/DeepSeek/…）一张卡片，可编辑 **API Key、默认模型、Base URL**；内置服务商与自定义端点同等可编辑。
- 编辑态有「测试」按钮，可**不保存**直接测当前填的配置是否连得通。
- 这里保存的模型配置是全系统模型的**单一真源**——各处的模型下拉都从这里动态生成。

### 5.2 AI 模型分配（给各角色指定模型）
系统把 LLM 使用点划成 3 个模块页签：
- **产业链分析**（拆解 / 供应商评估 / 交叉验证 / 圆桌…）
- **观察池**（催化剂 / 策略引擎 / 论点追踪 / 交易复盘 / 参数调优 / 深度分析…）
- **决策中心**（L1 宏观 / L2 组合 / L3 战术 / L4 执行 + 投委会各委员）

每个角色给它挑一个服务商 + 模型。**多模型角色**（右侧「多模型」标记）可配多个：
- 例如 **L1 宏观策略** 有两个槽：**宏观市场分析师 / 产业动向分析师**（用于宏观咨询的两位分析师，也是宏观策略的双模型交叉验证来源）；
- **瓶颈分析** 最多 3 个模型做交叉打分。

不配的角色会走默认服务商/模型兜底。

### 5.3 综合测试 & 自动推荐
- **综合测试**：批量测各服务商各模型的连通性/能力。
- **自动推荐**：系统按测试结果，为各模块角色推荐合适的模型，一键采纳。

> ✅ 最小可用：在 API Key 管理里配好至少一个服务商的 Key + 默认模型，再给「决策中心 / 产业链分析」的关键角色分配模型，就能开始用了。

## 6. 各功能使用指引

顶栏功能按业务流程排列：**产业链分析 » 观察池 » 决策中心 «» 模拟交易 | 系统配置中心**（管理员另有「用户管理中心」）。

### 6.1 产业链分析（首页）
系统入口，用三步法从产业方向挖瓶颈供应商。三种用法：

- **快速赛道分析**：卡片上有预设赛道（GPU/AI 算力、人形机器人…）。
  - 点赛道按钮直接开始分析；
  - 点赛道上的 **⋯ 按钮** 打开「配置赛道参数」详细设置，可改 **市场 / 产业方向 / 终端产品 / 主分析模型**（每个赛道各自保存），再「开始分析」。
- **自定义分析**：自己输入终端产品/行业，自定义拆解深度等参数后运行。
- **热点扫描（⚡）**：自动检测当前市场热点行业，作为分析切入点。

产出：产业链逐层拆解 → 瓶颈环节评分（稀缺性/不可替代性/供需缺口/定价权/技术壁垒）→ 候选供应商 → 多模型交叉验证后的 Top 推荐。可将看中的标的**加入观察池**。

### 6.2 观察池
你的自选股跟踪台。
- 股票分层（重点 focus / 普通 normal / 跟踪 track），可按市场（美股/A股/港股）筛选。
- 系统会为观察池股票采集**行情、新闻、情报、财报、龙虎榜**等数据。
- 点某只股票打开**详情抽屉**，含多个页签：基本信息 / 价格 / 新闻 / 资金 / 情报 / 策略 / **深度分析(UZI)**。
  - 深度分析(UZI)：多维数据 + 估值建模 + 杀猪盘/风险检测，产出个股深度报告。

### 6.3 决策中心
把观察池的个股信号，收敛成组合级、可执行的决策。四层 + 投委会：
- **L1 宏观策略**：判断当前市场环境（regime/风险偏好/板块轮动）。
  - 卡片上有 **💬 宏观咨询** 抽屉：先列出本轮获取的市场/新闻数据，两位分析师（宏观市场 / 产业动向）自动流式解读，你可实时提问，两人先各自作答再互评辩论；对话按市场滚动保存、可长期延续。
- **L2 组合策略**：目标仓位/板块配置/风控上限。
- **L3 战术计划**：个股级进出场计划。
- **L4 执行方案**：具体下单方案（数量/价格/方式），待你确认。
- **投委会**：多位 AI 委员从不同风格（风险/成长/价值/逆向）投票评审，可拦截高风险计划；支持用户质询。
- **催化剂**：跟踪影响标的的事件日历。

运行方式：可「日常决策」（轻量检查）或「全量刷新」（完整重跑 L1→L4）。

### 6.4 模拟交易
纸面账户，验证决策中心产出的策略表现（**非真实交易**）。5 个页签：
- **账户盈亏**：总价值/收益率/胜率等 + **权益曲线**（可切 30/90/180/365 天）+ 月度绩效/标的分布。
- **当前持仓**：持仓明细、成本、盈亏、占比；可展开单只的历史。
- **历史交易**：成交流水，可按 Ticker/方向筛选。
- **复盘记录**：交易复盘、经验卡片、交易反馈历史；可批量复盘。
- **账户操作**：存取资金、刷新市场价格、生成调优建议。

决策中心 L4 生成的执行计划，经你在待确认区确认后，会在模拟盘落地成交。

### 6.5 系统配置中心
见[第 5 节](#5-首次配置向导系统配置中心必做)——API Key、模型分配、测试、推荐都在这里。

### 6.6 用户管理中心（仅管理员）
管理员账户在顶栏可见「用户管理中心」图标，用于：
- 用户管理、邀请码管理、系统配置；
- 查看各用户的主要数据：产业链分析文件数量及分析参数、观察池股票数、模拟账户盈亏、持仓、AI 配置情况；
- 可**拷贝使用某用户的 AI 配置**到自己账户。

## 7. 命令行（CLI）用法

除网页外，还有几个命令行工具：

```bash
bottleneck-hunter serve     # 启动网页服务（主用法）
```

## 8. 常见问题排查

| 现象 | 原因 & 解决 |
|---|---|
| 分析提示「**无可用 LLM**」 | `.env` 没有可用 Key，或对应角色没分配模型。到 .env 填 Key，或在系统配置中心配 Key + 给角色（如 L1_macro）分配模型。 |
| **图表 / 权益曲线加载失败** | 前端图表库加载失败。本项目已把 echarts/d3/marked 本地化到 `static/vendor/`（国内 jsdelivr 等公共 CDN 不可达）。若你新增了公共 CDN 脚本会重现此问题——请改用本地 vendor。 |
| 注册收不到**验证码** | 未配置 SMTP 时验证码打印在**服务器日志**里，直接从日志取。要发真实邮件请在 .env 配 `SMTP_*`。 |
| 找不到 admin **密码** | 首次启动的随机密码只在服务器日志打印一次，搜日志「已创建默认管理员」。丢了可在无用户环境重新初始化，或用管理员工具重置。 |
| 页面各处宽度/图表异常 | 改动 CSS/前端后请**硬刷新**浏览器（Ctrl+Shift+R）清缓存。 |
| A 股数据 | A 股行情/龙虎榜等依赖 AKShare，美股依赖 yfinance，需要网络可达对应数据源。 |

## 9. 数据与隐私

- 数据存本地 **SQLite**（`data/` 目录），按用户隔离，各用户只见自己的数据（管理员可在用户管理中心聚合查看）。
- **模拟交易是纸面盘，不产生任何真实下单**。
- 系统输出为 AI 辅助分析，**不构成投资建议**，据此决策风险自负。
