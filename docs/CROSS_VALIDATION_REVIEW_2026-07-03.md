# 交叉评审（Cross-Validation）环节 · 专业审核意见

> **审核视角**：资深金融分析师 + LLM 评测/方法论
> **审核日期**：2026-07-03
> **审核对象**：`bottleneck_hunter/chain/cross_validation.py` + `chain/prompts/cv_*.md` + 其在 `graph.py`/`streaming/phases.py` 的接入
> **方法**：5 个专家视角并行评审 → 15 项发现逐条对抗性核实（0 条被驳回）→ 审查主管补盲；全部对照真实源码

---

## 一、先回答"每个模型拿到什么 prompt 和信息"

交叉评审对 `overall_score` 最高的 **前 10 个候选** 逐一验证（`cross_validation.py:335`），每个候选交给 N 个**用户在 AI 配置里为 `pipeline_cross_val` 角色选的模型**。4 个视角按序号轮询分配 `PERSPECTIVES[i % 4]`（`:164-168`）：

| 视角 | system prompt（人设） | user prompt 实际喂的信息（`_build_*`） |
|------|----------------------|------------------------------------|
| **财务审计员** financial | cv_financial_auditor.md：要求审"毛利率趋势/现金流/PE/负债率/造假嫌疑" | 只给 名称·代码·市场·市值·行业 + **3 个已算好的分数**（财务健康 X/10、估值 X/10、综合 X/10）。**零真实财务数字** |
| **产业链分析师** chain | cv_chain_analyst.md：要求审"竞争格局/市占率/CR3/议价权/替代风险" | 给 瓶颈节点名 + 3 个评分 + strengths/weaknesses 文本。**无 CR3/HHI/真实供应商竞争数据** |
| **情绪观察者** sentiment | cv_sentiment_observer.md：要求审"机构持仓/分析师评级/催化剂/做空" | 只给 综合评分 + strengths/weaknesses + "你对该公司的了解"。**无任何机构/分析师/催化剂真实数据** |
| **盲测对照** blind | cv_blind_test.md：要求"仅凭已有知识独立评估" | 只给 **公司名称** + 行业 + 市场。rubric 明写"完全不了解该公司→打 1-4 分" |

**聚合**（`:226-263`）：≥4 个分数→去掉最高最低取均值；否则取中位数。任一模型 `fatal_risk=true` → 共识分封顶 3.0。离群（|分-中位数|≥3）→ 记录标记"flagged"。

**接入决策**：`consensus_score≥5` → 进 `top_picks`（`graph.py:241`）；`≥7.5 pass / ≥5 concern` → 纯展示标签（`phases.py:556`）。**默认关闭**（`api.py:77 enable_cross_validation=False`），需手动开启。

## 二、核心判断（先给结论）

> **交叉评审的"人设/框架"设计是专业的，但"实现"从三个层面系统性地架空了它的四个目标——它当前更像一个昂贵的装饰环节，而非真正能查缺补漏、避免幻觉的关卡。**

一句话概括根本病灶:**它让"专家"在没有专家所需数据的情况下打分**——财务审计员看不到一个财务数字,产业链分析师拿不到 CR3,情绪观察者收不到任何机构数据。而这些真实数据**在上一环节(供应商评估)已经采集并存在于同一个 `SupplierScorecard` 对象里**,只是没被传进来。

---

## 三、逐目标评价

### 目标①「查缺补漏」——**基本无法达成**
交叉评审只拿到"前阶段算好的分数",没有拿到"前阶段用的原始数据"。它无法核对"财务健康 8/10 这个分打得对不对",因为它看不到毛利率、现金流、负债率。它只能对着一个 8 分反过来讲道理——这不是查缺补漏,是**在同一个结论上再包装一层**。真正的漏(比如某财务分基于缺失数据虚高)它发现不了,因为它和前阶段吃的是同一份(甚至更少的)信息。

### 目标②「发现被忽略的问题」——**基本无法达成 + 反向制造假信号**
- 系统里有 `institutional_holders`(机构持仓)、`analyst_ratings`(分析师评级)、`catalyst_tracking`(催化剂)、`smart_money`(聪明钱)、真实 `CR3/HHI` ——这些正是"被忽略的问题"最可能藏身之处,**却一个都没喂给交叉评审**。情绪观察者被要求盯机构动向,手里却没有任何机构数据(CONFIRMED,Critical)。
- 更糟:模型调用失败时**静默返回 5.0 并标"按中性处理"**(`:212-218`),而 5.0 恰好 ≥5 会把一个"其实调用失败"的标的抬进 top_picks——**失败被伪装成中性通过**,系统自己制造假信号。

### 目标③「避免幻觉/偏差」——**失效,甚至反而制造新幻觉**
- 财务/情绪/盲测三个视角因为没有真实数据,只能**凭训练知识编造**("我印象里这家公司…")或**锚定已给的分数**——这正是幻觉的温床,而非幻觉的解药。
- **盲测不盲**:暴露了公司名(`:117`)。对知名公司,模型带入训练偏见;对系统真正要找的**冷门卡脖子小盘股**,rubric"不了解就打 1-4 分"会**系统性误杀真正的璞玉**——这与产品"发现被忽视的优质供应商"的核心使命直接冲突。
- **伪多样性(最隐蔽)**:代码只按序号分视角,**完全不约束 provider**。用户若配 4 个同厂模型(如 4 个 deepseek 变体),得到的是同一训练数据的 4 次高相关采样——相关误差不会因取均值抵消,"共识"在统计上近乎自欺。而投委会(committee)是**刻意跨厂**的。

### 目标④「加强选择依据」——**几乎没有加强,只加了成本**
- consensus 只做 `≥5` 的**二元门 + 展示标签**,**不参与最终排序**(排序仍由 `final_score = quality^0.55 × alpha^0.45` 决定)。花 N×10 次付费推理(每次 120s 超时)收集的细粒度共识分,最后被压成一个 0/1 门槛——而这个门槛用现成的 `overall_score` 阈值零成本就能做。
- 唯一**有实际拦截力**的是 `fatal_risk` 封顶 3.0(会掉出 top_picks)。但它恰恰是**最没数据支撑**的一项——由模型凭世界知识判"财务造假/退市/技术替代",无 ST 标签、无真实财报、无市占率数据校验。**唯一起作用的机制,是最不可信的那个。**
- docstring 三重不符:自称"Weighted consensus"实则**无权重**(`ModelValidation.weight` 字段定义了却从不使用);自称"Outlier challenge round"实则**只标记不质询**(没有第二轮)。

## 四、与投委会(Committee)的职责重叠——最大的结构性盲区

`committee.py`(决策中心 L4 评审)做的是**同一件事**(多 LLM 从对立角度拷问投资逻辑),但每一项都做得更好:
- 真加权表决(`_member_weights` 取 `model_ratings` 历史校准权重)
- 2 轮辩论(委员看到他人意见后可改票)
- 分歧触发圆桌讨论、用户质询改票重算

交叉评审这一环**全无**这些(无权重、无辩论、无二轮、一次性打分)。所以真正该问的不是"怎么把交叉评审修好",而是:**既然投委会已经把"加权+辩论+离群解决"都做对了,交叉评审是否应该直接复用它的机制,或与之合并?** 两套并存意味着同一逻辑被两组模型分别拷问、口径(consensus_score vs approval_rate)不一致、双份 prompt 维护。

---

## 五、值得肯定的地方(勿因上述批评全盘否定)

1. **视角人设与 prompt 写得专业**:财务/产业链/情绪/盲测四个人设清晰、维度互补、盲点覆盖设计合理——地基是对的。
2. **去极值 + 离群标记框架合理**:防止单个离群分主导共识,统计思路正确。
3. **致命风险一票否决在原理上正确**:有实际拦截力(问题只在"判定依据无数据",而非机制本身)。
4. **只验证 top10**:成本/收益平衡合理。
5. **consensus 只做二元门**:代码没有过度声称它能精细排序——没有 over-promise 这一点是诚实的。

## 六、改进方向(按性价比排序,供决策;尚未动手)

**A. 立即止血(小改,高价值):把已有的真实数据喂进去**
根本问题是"数据管道断层"——数据在 `SupplierScorecard` 里(`financial_snapshot`/`smart_money`/`catalyst`)、在 `BottleneckReport` 里(真实 `CR3/HHI`,正是上次刚修的),只是没传进 `_build_*_prompt`。
- 财务视角注入 `financial_snapshot`(营收增速/毛利率/ROE/负债率/现金流 + trend)。
- 产业链视角注入真实 `CR3/HHI` + `cr3_source`(标明"实测 62%(akshare)"还是"LLM 估算")+ 候选在环节内的相对位置。
- 情绪视角注入 `smart_money`(机构持仓变动/龙虎榜/融资余额/做空比)+ `catalyst` + `analyst_rating`。
- 无数据时**显式标"数据缺失"**,禁止模型凭空编造。

**B. 修正会制造假信号的实现细节**
- 失败不要静默降级 5.0(会误进 top_picks),应标 `error` 且排除出门槛,或降到 <5。
- `fatal_risk` 需要证据链:能引用原始数据才允许一票否决;仅凭"世界知识"→降级为警告(降到 5 而非封顶 3),或要求 ≥2 视角共同确认。
- 盲测要么**真盲**(只给"某行业公司,市值约 X 亿",不给名),要么改名为"知识对照组"别再叫盲测。
- 约束 provider 多样性(至少警告同厂配置),否则目标③空转。

**C. 让它真正影响选择(否则不值这个成本)**
- 把 consensus 作为一个**因子并入 `final_score` 的重排序**(或对 concern/pass 做加减分),而非只做二元门。
- docstring 与实现对齐(去掉"weighted"或真的实现加权;"outlier challenge"要么实现二轮质询要么改名)。

**D. 战略层面(最该先想清楚)**
- **评估交叉评审与投委会是否合并**。若前阶段(产业链分析)确实需要一道独立质询关,建议直接**复用 committee 的加权/辩论机制**,而不是维护一套更弱的平行实现。

## 七、一句话审核意见

> **"交叉评审有一副好骨架(专业的四视角人设)和一个致命的空心(让专家在没有专家数据的情况下打分)。它现在既没能查缺补漏、也没能避免幻觉、更没能加强选择依据——因为它吃的是前一环节结论的'残渣'而非'原料',而真正的原料就躺在同一个对象里没被使用。先把已有真实数据接进去是立竿见影的止血;但更该回答的战略问题是:它与更成熟的投委会机制并存,是否还有独立存在的必要。"**
