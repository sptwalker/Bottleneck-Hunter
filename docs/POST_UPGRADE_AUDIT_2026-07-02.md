## 四、为何是 8.7 而非 9.0+（诚实披露）

按用户"每项真达标才算完成、不虚报"的要求，以下三项**机制已就绪但无法在当前条件下用数据证明达标**，故不计满分：

1. **回测/自进化的历史样本（-0.2）**：`realized_pnl`/`layer_performance`/`auto_reviews` 的写入链路已用 e2e 测试证明可产数据，但**生产库仍无真实卖出样本**（历史 3 笔全买入）。绩效统计的统计显著性要等真实交易积累——这是时间问题，非代码问题。诚实地说："机制就绪，等样本"。

2. **前视偏差（look-ahead bias）/ PIT 数据架构（-0.1）**：回测按 `created_at` 重放，但 market_snapshots/财务数据仍取"当前最新值"。真正的 point-in-time 快照是较大的数据工程，列为增强项。当前已通过"幻觉价禁成交"消除了执行环节的前视偏差。

3. **G-5 实盘隔离 / G-7 公司行动 / G-8 决策留痕（增强项）**：均为"接实盘前必做"或"可审计性增强"，当前纯模拟盘下风险未激活。按 YAGNI 原则**明确标记为增强项而非声称完成**——这正是避免重蹈 over-promise 覆辙的体现。

> **关键区别**：改造前的 4.6 分是"架构画了饼但没兑现"；现在的 8.7 分是"该兑现的都用数据兑现了，剩下的 0.3 是老实承认还需要真实样本/外部数据源，没有一处虚报"。
