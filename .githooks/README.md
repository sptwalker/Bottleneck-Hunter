# 首页更新记录自动生成

首页「🆕 更新历史」的内容来自仓库根的 `UPDATE_HISTORY.json`，现在**自动生成**——
不用再手动编辑那个文件。

## 怎么用（开发者）

在 commit message 里写一行以 `📢` 开头的**面向用户的白话**：

```
feat: 模型调度看板增强

技术描述随便写……

📢 配置中心新增调度看板 | 可以看到每个 AI 模型的成功率、是否正常，一目了然
```

- 格式：`📢 标题 | 详细说明`（`|` 可省略，省略则只有标题）
- 一条 commit 可写多行 `📢`，各成一条记录
- emoji 不便输入时，用 `[发布]` 等效
- **没有 `📢` 的 commit 不进更新记录**（纯内部重构不打扰用户）

提交后，`post-commit` 钩子自动把这条白话并入 `UPDATE_HISTORY.json` 并纳入本次提交。
推送到服务器 `docker build` 打包的就是最新的，首页自动显示。

## 启用一次（每台开发机）

git 钩子不随 clone 自动生效，需指向本仓库的 `.githooks/`：

```bash
git config core.hooksPath .githooks
```

Windows（Git Bash）同样这条命令。启用后无需再管。

## 手动生成（可选）

不依赖钩子，随时可手动跑：

```bash
python scripts/gen_update_history.py
```

幂等：从 git 历史的 `📢` + 现有条目合并去重（按 **日期+标题**），按日期倒序，现有手动条目不丢。

> 去重键是「日期+标题」：同一天同标题只留一条（保留现有值）；改了 commit 里的白话文案后，
> **在新的一天**提交才会作为新条目出现（旧条目仍保留，需要时手动删）。同一天内改文案不会覆盖，
> 想立即更新就手动编辑 `UPDATE_HISTORY.json` 删掉旧条目再跑脚本。
